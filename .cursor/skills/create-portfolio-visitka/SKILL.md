# Создание портфолио / визитки

Генерирует минималистичную одностраничную визитку с анимациями и адаптивной вёрсткой. Результат — один `index.html` со встроенным CSS. Без фреймворков и внешних ресурсов.

## Важно: без чужих персональных данных

**Никогда** не подставлять в сгенерированный результат реальные имя, email, ссылки или биографию автора навыка или третьих лиц. Использовать только:
- данные **текущего пользователя** из чата или
- явные **плейсхолдеры** (например «Имя Фамилия», «you@example.com», «your_username»), чтобы пользователь заменил их сам.

## Как запускать

### Вариант А — одним действием (плейсхолдеры)

Если пользователь просит «сайт-портфолио» или «визитку» без деталей:

1. Создать `index.html` по структуре и стилям из [template.html](template.html).
2. Использовать **только плейсхолдеры**: «Имя Фамилия», «Ваша должность или слоган», нейтральные абзацы «О себе», «you@example.com», «your_username» в ссылках.
3. После генерации написать, что нужно заменить (имя, должность, описание, текст «О себе», email, соцсети).

### Вариант Б — план и вопросы (данные пользователя)

Если пользователь хочет подставить свои данные — запросить по возможности одним сообщением:

- **Имя** (для h1 и подвала)
- **Должность или слоган** (подзаголовок в герое)
- **Краткое описание** в герое (1–2 предложения)
- **Блок «О себе»**: 2–3 абзаца
- **Email**
- **Ссылки** (например Telegram, GitHub, LinkedIn — URL и подписи)
- **По желанию**: цветовая схема (например «тёмная с фиолетовым», «бирюзовая», «янтарная») или оставить по умолчанию

Затем сгенерировать `index.html` с **его** данными. Не подставлять и не хардкодить данные других людей.

## Спецификация страницы

- **Файл**: один `index.html`; весь CSS в `<style>` в `<head>`. Без внешних CSS/JS/шрифтов.
- **Секции**:
  1. **Hero**: `h1` (имя), `.title` (должность/слоган), `.description` (краткий текст).
  2. **О себе**: `h2` «О себе» (или на языке пользователя), 2–3 абзаца.
  3. **Контакты**: `h2` «Контакты», список ссылок (email `mailto:`, Telegram, GitHub, LinkedIn и др.).
  4. **Подвал**: например «© 2025 · [Имя]».
- **Оформление**: минимализм. Тёмный фон с градиентом, один акцентный цвет (например бирюзовый или фиолетовый). Цвета через CSS-переменные.
- **Анимации**: клюфреймы `fadeInUp`; применить к герою, секциям, ссылкам контактов, подвалу с разными `animation-delay`.
- **Ссылки**: при наведении — например `translateY(-2px)`, смена рамки/фона, по желанию лёгкий `box-shadow`. `:focus-visible` для доступности.
- **Адаптив**: `@media (max-width: 600px)` — уменьшить отступы, контакты в колонку, кнопки на всю ширину при необходимости.
- **Язык**: по умолчанию `lang="ru"` и русские подписи, если пользователь не попросил другой язык.

## Шаблон

Использовать [template.html](template.html) как эталон. В нём **только плейсхолдеры**. При варианте Б подставлять данные текущего пользователя; при варианте А оставлять плейсхолдеры и напомнить пользователю заменить их.

## Как делиться навыком

Чтобы передать навык другому: скопировать папку `create-portfolio-visitka` (с `SKILL.md` и `template.html`) в папку навыков Cursor:

- **В рамках проекта**: `.cursor/skills/create-portfolio-visitka/` в репозитории
- **Для всех проектов**: `~/.cursor/skills/create-portfolio-visitka/`

После этого можно просить Cursor «сделай визитку» или «create a portfolio visítka» и получать такую же страницу со своими данными или плейсхолдерами.
